ARCHIVE POLICY

Preservation

The live site is a derived exhibit. The archive is the receipt trail.

‹ HomeReceiptsNegativesMarkdown

What gets frozen

  • Markdown doctrine and receipts.
  • Static HTML/CSS/JS public exhibit.
  • Structured JSON index.
  • SVG assets and generator scripts.
  • SHA-256 manifests.

Export command

node scripts/export-static-package.mjs

The command creates exports/<date>-boring-receipts-lab/ with manifest.json and SHA256SUMS.

Boundary

A package preserves the proof trail and command shape. It does not bundle vendor binaries, model weights or external datasets. Those dependencies remain named in each receipt and should be mirrored or hashed when license and storage constraints allow.

Genealogy

The lab began as a refusal of speculative drift: crypto detour → Waffle/TurboQuant validation → boring receipts. Naming the genealogy keeps the archive from becoming a decontextualized benchmark shelf.

Read GENEALOGY.md and PRESERVATION.md.
